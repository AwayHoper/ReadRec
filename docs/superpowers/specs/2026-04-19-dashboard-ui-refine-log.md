# 首页展示优化轻量日志

日期：2026-04-19

## 目标

- 首页从“模块堆叠”收敛为“克制的学习驾驶舱”
- 减少重复文案，优先通过图形和布局表达信息
- 保留首页作为开始学习入口的清晰主路径

## 设计方向

- 整页压缩为 `3` 个主区块，而不是连续堆叠的大卡片
- 风格偏“学习驾驶舱”，减少解释型文案和独立功能卡
- 保留必要数据，但弱化低优先级信息的视觉权重

## 页面结构

### 1. 顶部主区

- 使用一个宽 Hero 承担首页主视觉
- 左侧展示：
  - 今日状态
  - 核心数字（新词、复习词、今日已学）
  - 主按钮（开始学习 / 再学一轮）
  - 一行鼓励文案小字
- 右侧展示：
  - 掌握度环形图
  - 右上角 `i` 提示

### 2. 学习日历区

- 使用真实日历组件替代当前块状日历
- 每个日期的背景深浅表示学习强度
- 顶部只保留一行轻量历史摘要，不再单独拆卡

### 3. 今日摘要区

- 保留紧凑摘要面板
- 未完成时展示：新词 / 复习词 / 今日计划
- 已完成时展示：今日已学习 / 已完成轮次 / 当前状态
- 最近完成记录并入摘要底部，不再单独做大卡片

## 信息展示约定

- 鼓励文案不再单独占一个模块，只作为主按钮下的一行辅助信息
- 掌握度定义不直接展开显示，通过 hover `i` 提示查看：
  - 熟悉：已复习的单词
  - 模糊：已学但还未复习的词
  - 陌生：未进行学习的单词
- 首页优先表达“今天该做什么”，其次再表达“历史学了多少”

## 实现约定

- 本次作为展示优化迭代处理，不单独展开完整 spec 和 implementation plan
- 后续实现阶段直接按本日志进行页面精简、布局重组和视觉收敛
