# 首页学习中心设计（v0.6.0）

## 背景

当前首页仅展示当前词库、session 状态与一个固定的开始学习入口，无法承担“今日学习中枢”的职责。随着计划模块与真实词库接入完成，首页需要升级为围绕“今天学什么、学了多少、下一步做什么”展开的学习中心。

本次 v0.6.0 聚焦首页改造，不调整计划页的方案选择逻辑，也不重做学习流程本身。

## 目标

- 首页成为用户进入产品后的第一学习入口
- 清晰展示今日学习状态、词库掌握情况与历史打卡信息
- 支持“开始学习”与“再学一轮”两种入口状态切换
- 让首页统计口径与当前抽词、计划、学习流程保持一致
- 将“再学一轮”规则同步写入业务与技术文档

## 非目标

- 不在本次实现签到进度排行榜、周/月排行榜
- 不在本次实现首页可视化主题重构以外的全站 UI overhaul
- 不改变当前计划页 `1:1 / 1:2 / 1:3 / 1:4` 与 `5 / 10 / 20 / 40` 的核心规则
- 不引入额外的学习流程轮次

## 用户故事

- 作为已配置学习计划的用户，我希望一进入首页就知道今天要学多少词，方便立刻开始。
- 作为今天已经完成一轮学习的用户，我希望首页明确告诉我“今日计划已完成”，并鼓励我再学一轮。
- 作为持续学习的用户，我希望看到自己的打卡痕迹、连续学习情况和当前词库掌握程度。

## 产品设计

### 1. 首页整体结构

首页改造为四块内容，自上而下排列：

1. 今日学习主卡
2. 学习进度与词库掌握情况双栏区
3. 日历打卡板
4. 鼓励文案与历史学习摘要

页面主职责为“展示当前学习状态 + 提供开始下一步学习的入口”，不再承载词库管理或计划编辑入口。

### 2. 今日学习主卡

主卡是首页最核心的区域，包含：

- 当前词库名称
- 今日学习状态摘要
- 主按钮：`开始学习` 或 `再学一轮`

#### 2.1 状态切换规则

- 若用户当天还没有完成任何一批学习：
  - 展示“待完成：新词 X 个，复习词 Y 个”
  - 按钮显示 `开始学习`
- 若用户当天已经完整完成过至少 1 个批次：
  - 展示“已完成：今日已学习 Z 个”
  - 按钮显示 `再学一轮`

这里的“今日计划已完成”以“今天至少完成过 1 个批次”为准，而不是追踪所有后续额外批次的完成状态。

#### 2.2 数据口径

- `X / Y` 来自当前词库当前学习计划推导出的首批学习目标
- `Z` 为“今天所有已完成批次中，已完成完整学习流程的去重单词数”
- 未完成完整学习流程的单词不计入 `Z`

#### 2.3 再学一轮规则

- 用户完成今天首批学习后，首页按钮变为 `再学一轮`
- 点击后继续沿用：
  - 当前激活词库
  - 当前学习计划比例
  - 当前抽词规则
- 系统重新抽取下一批学习词，用户当天可无限次开启新批次
- 该规则需要同步写入业务文档和技术文档

### 3. 今日学习进度区

首页中部左侧展示今日学习进度，承担“计划状态说明”的职责。

#### 3.1 未完成首批时

展示：

- 待完成新词数
- 待完成复习词数
- 当前计划的每日总量

文案强调“今天要完成的首批计划”。

#### 3.2 已完成首批时

展示：

- 今日已完成去重词数
- 今日已完成批次数
- 当前状态标签：`今日计划已完成`

若今日完成批次数大于等于 2，则首页鼓励文案切换到“超额完成”状态，但这一块的主状态仍保持“今日计划已完成”。

### 4. 当前词库掌握情况

首页中部右侧展示当前激活词库的掌握情况，使用环形图或等价的三段比例图。

分类定义如下：

- 熟悉：已进入复习词集合的单词，即已经历学习并完成复习阶段的词
- 模糊：已学但未进入复习掌握状态的词，即经历过学习流程但尚未被视为“熟悉”的词
- 陌生：当前词库中从未进行过学习流程的词

展示内容包括：

- 三类数量
- 三类占比
- 当前词库总词数

首页只展示当前激活词库的掌握情况，不跨词库汇总。

### 5. 日历打卡板

首页下方展示一个按天分布的打卡面板，视觉参考 GitHub contributions。

#### 5.1 上色规则

- 当天存在已完成学习记录，则该日期有背景色
- 颜色深浅按“当天完成完整学习流程的去重单词数”分级
- 未学习日期不着色

#### 5.2 打卡统计

日历区域同时展示：

- 累计打卡天数
- 连续打卡天数
- 再打卡多少天即可完成当前词库计划
- 预计完成日期

其中：

- “累计打卡天数”表示历史上有完成学习记录的自然日数量
- “连续打卡天数”表示截至今天向前连续有完成学习记录的天数
- “再打卡多少天即可完成当前词库计划”基于当前词库剩余新词量和当前计划新词数计算
- “预计完成日期”与计划页保持一致，基于当前方案推导

### 6. 鼓励文案

首页展示一段与今日状态联动的鼓励文案，分为三档：

- 未完成今日首批计划：励志鼓励型
- 已完成今日首批计划：表扬型，并鼓励继续学习
- 超额完成：高度表扬型

#### 6.1 状态判定

- 未完成：当天尚未完成任何批次
- 已完成：当天完成批次数为 1
- 超额完成：当天完成批次数大于等于 2

文案可先使用规则映射生成，后续如需 AIGC 个性化文案再单独迭代。

### 7. 历史学习摘要

首页可以在鼓励文案区域或其下方展示简洁的历史摘要，范围控制在本次 MVP 可稳定提供的数据：

- 最近一次学习日期
- 最近一次完成的批次词量
- 当前活跃词库名称

如果当前已有更强的首页密度压力，该块可以以简短文本形式出现，不单独做重卡片。

## 后端设计

### 1. 首页聚合接口

建议新增首页聚合接口，由后端一次性返回首页所需核心信息，避免前端拼装过多请求。

候选接口：

- `GET /dashboard/home`

返回内容建议至少包含：

- 当前激活词库信息
- 当前计划摘要
- 今日主状态：`pending` / `completed`
- 今日首批目标新词数、复习词数、总量
- 今日已完成去重词数
- 今日已完成批次数
- 首页按钮状态：`start` / `continue`
- 词库掌握情况三分类数量
- 打卡日历数据
- 累计打卡天数
- 连续打卡天数
- 剩余打卡天数与预计完成日期
- 鼓励文案状态：`encourage` / `praise` / `celebrate`
- 历史学习摘要

### 2. 今日状态判定

首页今日状态以“是否完成过至少一个批次”为准：

- 若今天没有任何已完成批次，则为 `pending`
- 若今天至少存在一个已完成批次，则为 `completed`

这里不因为后续额外开启的新批次尚未完成而回退为 `pending`。

### 3. 去重学习数统计

“今日已学习 X 个”采用如下规则：

- 统计今天所有已完成批次中完成完整学习流程的单词
- 以单词去重后计数
- 未完成整轮流程的单词不纳入统计

### 4. 掌握情况判定

本次保持与现有学习状态模型兼容，不额外引入新的评分体系。

建议口径：

- 熟悉：已复习过的词
- 模糊：已学但未复习的词
- 陌生：从未学习过的词

若现有数据模型无法直接区分“已复习”与“已学未复习”，则需要在实现计划中补充映射规则或最小字段增强，但不在本 spec 中重构完整掌握度体系。

### 5. 再学一轮接口行为

当前学习入口若已存在“获取今日 session / 创建今日 session”的逻辑，需要扩展为支持同一天多批次：

- 若当天尚未完成首批，则主按钮进入当前首批学习
- 若当天已完成至少一批，点击 `再学一轮` 需要创建新的批次 session
- 新批次继续沿用当前词库与当前计划比例
- 抽词规则与现有每日抽词规则一致

实现计划中需明确：

- 同一天多批次 session 的建模方式
- 当前“唯一今日 session”约束是否要放宽或改造
- 前端如何路由到“当前进行中的批次”

## 前端设计

### 1. 页面结构

首页采用学习中心布局，优先突出状态与行动按钮。

建议视觉层级：

- 顶部一张强信息主卡
- 中部两张等权信息卡
- 下部一块横向打卡板
- 最后是鼓励文案与历史摘要

### 2. 按钮交互

- `开始学习`：进入今日首批学习
- `再学一轮`：进入当日下一批学习

按钮文案仅由首页聚合状态决定，前端不自行推断复杂业务逻辑。

### 3. 空状态与异常状态

- 若用户未配置学习计划：
  - 首页展示引导去计划页设置计划
- 若当前没有激活词库：
  - 首页展示引导选择词库
- 若首页聚合接口异常：
  - 首页展示可恢复的错误提示与重试入口

## 数据与公式约定

### 1. 今日首批目标

由当前激活词库的学习计划推导：

- 新词目标：由方案卡片选中的新词数决定
- 复习目标：新词目标乘以复习倍数
- 总量：新词目标 + 复习目标

### 2. 剩余完成天数

沿用计划页公式：

- `剩余天数 = ceil((总词数 - 已完成词数) / 每日新词数)`

这里的“已完成词数”指当前词库中已进入学习完成集合的词数。

### 3. 打卡强度颜色

本次先采用固定分档，不做连续渐变算法。建议四档：

- `0`：无颜色
- `1-10`
- `11-25`
- `26-50`
- `51+`

具体阈值可在实现时根据 UI 效果微调，但必须在技术文档中最终落定。

## 测试与验收

### 1. 功能验收

- 未开始学习时，首页显示首批待完成新词数与复习词数
- 完成首批后，首页切换为“今日计划已完成”并显示 `再学一轮`
- 同一天完成多批次后，首页仍保持完成态，但鼓励文案切换为超额完成
- 掌握情况图数据与当前词库状态一致
- 日历打卡颜色能反映当天学习强度
- 未配置计划与未选词库时展示正确引导

### 2. 技术验收

- 首页首屏不依赖多次串行请求才能渲染核心状态
- 同一天多批次不会破坏原有学习流程数据读取
- 文档同步更新业务口径、技术口径与“再学一轮”规则

## 风险与注意事项

- 当前系统如果仍然以“每天只存在一个 session”为前提，则“再学一轮”会触发模型与接口改造
- “熟悉 / 模糊 / 陌生”的精确定义依赖学习状态建模，若现有字段不足，需要在实现计划中明确最小改造方案
- 日历打卡板需要注意时区口径统一，避免跨天误判
- 首页聚合接口需要避免与计划页、学习页统计口径不一致
